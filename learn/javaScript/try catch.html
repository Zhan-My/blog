# try catch 捕获异常

一句话概括

异步方法无法捕捉到，能捕捉到的异常必须是线程执行已经进入try catch 但 try catch 未执行完的时候抛出来的。

# 线程执行处于try catch之前

  • 代码报错的时候，线程未进入try catch，那么无法捕捉异常，比如语法异常（syntaxError），语法检查阶段就报错
try {
    fn.;
} catch (err) {
    console.log(err);
}
// Uncaught SyntaxError: Unexpected token ';'

# 线程执行处于try catch之中

  • 代码报错的时候，线程未进入try catch，那么无法捕捉异常，比如语法异常（syntaxError），语法检查阶段就报错
try {
    function fn() {
        a.b
    };
    fn();
} catch (err) {
    console.log(err);
}
// ReferenceError: a is not defined
  • 方法在执行之前定义也是可以捕捉到异常的
function fn() {
    a.b
};
try {
    fn();
} catch (err) {
    console.log(err);
}
// ReferenceError: a is not defined

# 线程执行处于try catch之后

  • 代码报错的时候，线程未进入try catch，那么无法捕捉异常，比如语法异常（syntaxError），语法检查阶段就报错
try {
    function fn() {
        a.b
    };
} catch (err) {
    console.log(err);
}
fn();
// Uncaught ReferenceError: a is not defined
  • 再如延迟函数
try {
    setTimeout(() => {
        a.b
    })
} catch (err) {
    console.log(err);
}
// Uncaught ReferenceError: a is not defined

# Promise 没异常

try {
    new Promise((resole, reject) => {
        a.b
    }).then(res => {
        console.log(res);
    })
} catch (err) {
    console.log(err);
}
// Uncaught (in promise) ReferenceError: a is not defined
