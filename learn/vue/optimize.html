# vue页面优化

# 1. v-if和v-show区分场景使用

  • 频繁切换，则需要提前渲染好，控制display显隐即可，适用v-show
  • 不涉及频繁切换，则可以以使用v-if

# 2. v-for遍历，加 :key="xxx"，提高性能（diff算法进行父子节点更新的时候基于该key）

# 3. 针对纯展示列表，axios请求过后进行this.list = Object.freeze(data)，冻结对象，禁止双向绑定修改对象，减少一定的性能消耗

# 4. 使用vant等UI库，使用babel-plugin-import、babel-plugin-component等插件按需引入，而不是使用全部引入，减少包体积

# 5. 大量图片使用vue-lazyload懒加载等插件

# 6. 路由懒加载，也是常规习惯

# 7. 长列表渲染

  • vue-virtual-scroll-list (opens new window)
  • vue-virtual-scroller (opens new window)

# 8. 骨架/呼吸屏，提升用户体验

# 9. SSR服务端渲染，相当于服务端直接返回html页面，提升首屏加载速度

# 10. 架构层面，前端微服务，之前公司使用docker+caas平台，特别对于APP内嵌的情况，分模块各自部署在独立的微服务，减轻服务器压力，同时可以无感更新，提升用户体验

# 11.多页面

  • 基于vueCli的移动端多页面脚手架，整理了结构目录和URL参数验证以及常用的方法封装，开箱即用 (opens new window)
  • 使用多页面取代SPA，加快首屏加载速度
